Layout β

This component is still in beta and should in time replace the existing templates.

A migration period will allow to switch at its own pace once the component is officially published, then its use will be made mandatory and the old templates will disappear.

Its benefits

  • it no longer mixes layout and presentation of embedded components;
  • only one or two scrollbars can appear and they are never adjacent to each other or stacked on top of the content;
  • only the height of the banner is fixed, everything else adapts according to the content;
  • the code is compact, IE 11 compatible (with graceful degradations) semantic, responsive and accessible.

Its limitations

  • When one of the columns has a content higher than the main content, the main content scrolls with the columns.

Structure

<div class="layout"> <header class="layout-banner"> … </header> <nav class="layout-navside"> <button id="navsideToggle" aria-expanded="false" class="layout-navside-toggle">Menu</button> <div class="layout-navside-content"> <div class="layout-navside-content-inner"> … </div> <footer class="layout-navside-footer"> … </footer> </div> </nav> <main class="layout-content"> <div class="layout-content-headers"> <header class="layout-content-headers-header"> <div class="layout-content-container"> … </div> </header> <header class="layout-content-headers-subheader"> <div class="layout-content-container"> … </div> </header> </div> <div class="layout-content-inner"> <div class="layout-content-container"> … </div> </div> <footer class="layout-content-footer"> <div class="layout-content-container"> … </div> </footer> </main> <aside class="layout-aside"> <div class="layout-aside-content"> <div class="layout-aside-content-inner"> … </div> <footer class="layout-aside-footer"> … </footer> </div> </aside> </div>

Minimal

These parts of the layout are optional, you can directly remove them from the DOM if necessary.

  • layout-content-headers-header
  • layout-content-headers-subheader
  • layout-content-footer
  • layout-navside-footer
  • layout-aside-footer
  • layout-aside (you must also add a .mod-noAside class for this one)

Tip: we can also avoid the layout-container if we want to.

Modifiers

You can add these modifiers to the .layout, they change the default behavior of the structure:

  • .mod-noScrollbarNavside to remove the scrollbar from the navigation menu and make the main scrollbar of the page inherit its management;
  • .mod-noNavsideCompact to double the width of the navigation menu;
  • .mod-noAsideAfter to put secondary content before main content;
  • .mod-noStickyContentHeader to disable the sticky behavior of the header;
  • .mod-noStickyContentSubheader to disable the sticky behavior of the subheader;
  • .mod-noStickyNavsideFooter to disable the sticky behavior on the footer of the navigation menu;
  • .mod-noStickyAsideFooter to disable the sticky behavior on the footer of the aside menu;
  • .mod-noStickyContentFooter to disable the sticky behavior on the footer of the main content;

Important: you have to delete the headers container if you only want the header or the subheader sticky.
