## Cas d'usage

#### **✅  Utilisons le Tooltip pour...**

* Décrire la fonction d'un élément lorsqu'elle peut être ambiguë. Exemples : explications d'un sigle, d’une formule de calcul, nom d’un collaborateur concerné.
* Expliquer la raison de l’état inactif d’un élément interactif.
* Afficher l’intégralité d’un intitulé tronqué ou abrégé, à condition que l'intégralité n'excède pas 2 lignes.

#### **❌  Ne l'utilisons pas pour...**

* Fournir des informations que l'utilisateur doit connaître ou dont il doit se souvenir.
* Afficher un texte très long (plus de 2 lignes). 
* On privilégie alors le [Popover](https://prisme.lucca.io/94310e217/p/129fae).

### **Tooltip ou Popover ?**

Le [Popover](https://prisme.lucca.io/94310e217/p/129fae) est recommandé pour une description plus longue, du texte formaté et des images. Ou des liens facultatifs.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons le Popover lorsque le contenu est riche et/ou nécessite d'être mis en forme. |
| Don't |  |   | Le Tooltip ne peut contenir que du texte. Ne l'utilisons pas pour mettre en forme du contenu. |

# Content

## Contenu et rédaction

* Rédigeons des messages toujours plus brefs et concis, avec les principaux mots-clefs apparaissant en premier. **Une ligne dans l’idéal,** deux maximum.
* **Terminons les phrases complètes (sujet, verbe, objet) par un point**. Nous n’utilisons jamais de point d’exclamation ou d’interrogation.
* Le contenu peut expliquer l’action qui suivra un clic.
* Ne transmettons pas de mises à jour d'état, ni de commentaires. D'autres composants sont mieux adaptés à la communication de ce type d'informations.
* Le Tooltip porte uniquement sur l’objet qu’il est censé expliciter. N’ajoutons aucune information en plus.
* Ne répétons pas l’information contenue dans l’interface. Un bon Tooltip révèle dans l’idéal quelque chose qu'un utilisateur n'aurait pas pu apprendre par lui-même.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Rédigeons des messages toujours plus brefs et concis, avec les principaux mots-clefs apparaissant en premier. |
| Don't |  |   | Ne rédigeons pas de messages trop long. Cela rend la lecture très compliqué pour l'utilisateur. |
