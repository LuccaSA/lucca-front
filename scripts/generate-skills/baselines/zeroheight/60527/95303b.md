# Content

## Contenu et rédaction

### La bonne information, au bon moment

Le Link tient compte de la [loi de Fitts](https://prisme.lucca.io/94310e217/p/70492f-regles-dor/t/page-70492f-49497506-36). Le texte seul du Link permet d’en comprendre la fonction et la destination.

Un lien peut être placé au sein d’une phrase ou à la suite de celle-ci, selon ce qui est le plus clair et pertinent pour l’utilisateur. L’essentiel est que le lien soit explicite, compréhensible et facilement identifiable dans son contexte.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Fournissons un texte explicite et spécifique qui informe l'utilisateur de la fonction et de la destination du lien. |
| Don't |  |   | N'utilisons pas un texte de lien générique et peu informatif obligeant l'utilisateur à lire le contexte autour pour comprendre la destination. |

### Utilisation de l’icône arrow-external

Nous ajoutons une icône `arrow-external` après le label du lien pour informer l’utilisateur qu’il sera redirigé vers un nouveau contexte de navigation (onglet ou fenêtre).

* Un lien renvoyant vers un autre logiciel Lucca, peut, ou non, se faire dans un nouveau contexte.
* Un lien renvoyant vers un domaine extérieur s'ouvre systématiquement dans un nouveau contexte de navigation.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons l'icône arrow-external pour indiquer un changement de contexte de navigation. |
| Don't |  |   | Dans un tableau ou une liste dense, n'ajoutons pas l'icône arrow-external aux liens même s'ils ouvrent un nouveau contexte de navigation. Cela surcharge inutilement le contenu. |

### Vocabulaire et ponctuation

Nous écartons les verbes « voir » (remplacer par « afficher », « consulter »), « aller (sur) », « retourner (vers) » : ils peuvent toujours être remplacés par des [verbes](https://prisme.lucca.io/94310e217/p/18765d) plus précis.

Nous évitons les formulations comme « En savoir plus », « Plus d'informations », « Besoin d'aide ? » : elles peuvent toujours être remplacées par des formulations plus précises.

Si le Link se trouve à la fin d'une phrase ou d'un paragraphe, assurons-nous que l’intitulé du Link n'inclut pas de point, ni de point d’exclamation ou d’interrogation.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons des liens clairs, concis et actionnables qui décrivent précisément l'action effectuée et la destination. |
| Don't |  |   | Nous écartons les verbes « voir », « aller (sur) », « retourner (vers) » : ils peuvent toujours être remplacés par des verbes plus précis. |

### Link ou Button ?

Le Link n’a pas le rôle d’un [Button](https://prisme.lucca.io/94310e217/p/098404) : le Button est destiné à effectuer des actions précises, et non à rediriger les utilisateurs vers une autre page. L'action du Button se produit sur la même page.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons des Button pour effectuer des actions précises. |
| Don't |  |   | N’utilisons pas de Link pour représenter des actions, même si cela peut sembler plus esthétique ou dense. |

### Navigation interne et externe 

On privilégiera l'utilisation du routing Angular pour la navigation **interne** à l'application. Dans ce cas, la route est à passer directement via `luLink`.

```html
<a luLink="./#example">…</a>
```

Dans le cas d'une navigation **externe** à l'application, l'URL est à renseigner via la propriété `href`.

```html
<a luLink href="https://www.lucca.fr">…</a>
```

Dans les deux cas, l'ouverture du lien peut se faire dans la même page ou dans une nouvelle page. Dans le second cas, on utilisera la propriété `external`.

### Sécurité

Pour des raisons de sécurité il convient de rajouter à nos liens externes, les attributs `rel="noopener noreferrer"`. Ceci évite au navigateur d'ouvrir la ressource cible en donnant au nouveau contexte de navigation l'accès au document précédent. Pour plus de précisions : [https://developer.mozilla.org/fr/docs/Web/HTML/Attributes/rel/noopener](https://developer.mozilla.org/fr/docs/Web/HTML/Attributes/rel/noopener)
