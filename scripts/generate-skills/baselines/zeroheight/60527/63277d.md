# Content

### Guillemets

#### Premier niveau

* En français, on utilise les guillemets en chevrons.

Exemple : « ma phrase »

* On utilise aussi des espaces insécables entre les guillemets, afin d'interdire un saut de ligne entre les caractères.

<callout background="2">

Raccourci pour les guillemets

Windows `alt + 0171` et `alt + 0187`

macOS `option + 7` et `option + maj + 7`

</callout>

<callout background="2">

Raccourci pour une espace insécable

Windows `alt + 0160` ou `alt + espace`

macOS `option + espace`

</callout>

* En anglais, on utilise les guillemets en apostrophe.

Exemple : “my sentence”

<callout background="2">

Windows `alt + 0147` et `alt + 0148`

macOS `option + 3` et `option + maj + 3`

</callout>

* En espagnol, on utilise les mêmes guillemets qu'en français, mais sans espace insécable. 

Exemple : «mi sentencia»

<callout background="2">

Windows `alt + 0171` et `alt + 0187`

macOS `option + 7` et `option + maj + 7`

</callout>

* En allemand, on utilise les guillemets en apostrophes tournées.

Exemple : „mein Satz“

<callout background="2">

Windows `alt + 0132` et `alt + 0147`

macOS `option + maj + 2` et `option + 3`

</callout>

#### Second niveau 

Pour une citation insérée dans une autre citation, on utilise des guillemets différents.

* En français, on utilise les guillemets en chevrons simple.

Exemple : ‹ ma phrase ›

* Il faut encore et toujours utiliser des espaces insécables entre les guillemets, afin d'interdire un saut de ligne entre les caractères.

<callout background="2">

Raccourci pour les guillemets

Windows `alt + 0139` et `alt + 0155`

macOS `option + w` et `option + maj + w`

</callout>

<callout background="2">

Raccourci pour une espace insécable

Windows `alt + 0160` ou `alt + espace`

macOS `option + espace`

</callout>

* En anglais, on utilise les guillemets simples en apostrophe.

Exemple : ‘my sentence’

<callout background="2">

Windows `alt + 0145` et `alt + 0146`

macOS `option + 4` et `option + maj + 4`

</callout>

* En espagnol, on utilise les mêmes guillemets qu'en français, mais sans espace insécable. 

Exemple : ‹mi sentencia›

<callout background="2">

Windows `alt + 0139` et `alt + 0155`

macOS `option + w` et `option + maj + w`

</callout>

* En allemand, on utilise les guillemets simples.

Exemple : ‚my sentence‘

<callout background="2">

Windows `alt + 0130` et `alt + 0145`

macOS `option + maj + r` et `option + 4`

</callout>

### Apostrophes et guillemets typographiques

L‘apostrophe droite, ou apostrophe dactylographique, ne doit pas être utilisée dans des textes destinés à être lus. 

On utilise à la place les apostrophes typographiques.

| ✅ | ❌ |
| --- | --- |
| Aujourd’hui | Aujourd'hui |

<callout background="2">

Windows `alt + 0146`

macOS `option + shift + '`

</callout>

### Espaces

* La virgule et le point sont suivis d’un espace après, mais pas avant.
* La plupart des autres signes de ponctuation (point-virgule, deux-points, etc.) sont précédés d’une espace insécable et suivis d’un espace.
* Les parenthèses sont précédées et suivies d'une espace, mais pas d'espace à l‘intérieur.

| ✅ | ❌ |
| --- | --- |
| Voulez-vous enregistrer ? | Voulez-vous enregistrer? |

<callout background="2">

En anglais, ces signes de ponctuation ne sont pas précédés d’une espace.

</callout>

* Pour les parenthèses ou crochets, la ponctuation est incluse si la phrase est complète.

| ✅ | ❌ |
| --- | --- |
| [Il tend sa main.] Prends ma main avant de tomber. | [Il tend sa main]. Prends ma main avant de tomber. |

<callout background="2">

Raccourci pour une espace insécable

Windows `alt + 0160` ou `alt + espace`

macOS `option + espace`

</callout>

### Accent sur les majuscules, ou capitalisation

* Les majuscules doivent être accentuées. 

Oublier un accent constitue une faute d’orthographe, mais **c’est aussi une source d’inaccessibilité**, notamment pour les personnes dyslexiques.

* Les diacritiques, signes accompagnant une lettre ou un graphème pour en modifier le sens ou la prononciation, sont conservés (Ç).

| ✅ | ❌ |
| --- | --- |
| Éditer la campagne | Editer la campagne |

### Esperluette (&)

**À utiliser uniquement pour :**

* Les noms d'entreprises : *Fromagerie Lya & Mona, Bernard&Sons Poolrooms.*
* Les expressions établies : *R&D* (Recherche et développement)

**Privilégions toujours « et » ou la virgule** dans nos intitulés.

Nous n'utilisons pas l'esperluette entre deux mots dans les interfaces, pour plusieurs raisons :

* Elle rend les textes moins lisibles.
* Elle pose des problèmes de traduction automatique.
* Elle n'est pas adaptée à tous les contextes de localisation.

| ✅ | ❌ |
| --- | --- |
| Authentification et SSO | Authentification & SSO |
| Essence, station-service | Essence & Station-service |

### Titres toujours sans point 

Le titre, ou *[title](https://prisme.lucca.io/94310e217/p/021fef-title--text/b/4497c7)* d’une page, d’une section, d’une infographie, est par définition séparé du reste du texte. Même quand il est constitué d’une phrase complète.

* Nous évitons l'utilisation du point d'exclamation et des [deux-points](https://prisme.lucca.io/94310e217/p/63277d-ponctuation-typographie/t/63115d) à la fin des titres.

**Nous mettons une majuscule en début de titre.** **Nous ne mettons pas de point en fin de titre.**

* En français, **il s’agit d’un usage**, non d’une règle. Il nous faut choisir la même marche à suivre pour tous nos logiciels.

#### Exceptions

* Le titre se confond avec le reste du contenu qui suit. Si le point n’est pas mis, on peut penser qu’il y a une faute.
* Le titre s’achève sur un point d’interrogation. 

*Post LeMonde.fr sur* *[la pratique des titres sans point](https://www.lemonde.fr/blog/correcteurs/2009/09/30/un-point-sest-fait-la-malle/)**.*

### Texte tronqué, points de suspension

Pour l'aperçu d’un texte ou d'un label long, nous pouvons tronquer le texte en utilisant le caractère “…”. 

Ce caractère ne doit pas être confondu avec une succession de trois points. 

Il n’y a pas d’espacement avant ce signe, mais un espacement après.

Nous pouvons utiliser une balise title.

Dans tous les cas, nous ne devons jamais tronquer un texte dont la lecture complète est primordiale pour la compréhension.

| ✅ | ❌ |
| --- | --- |
| Chargement des données… | Chargement des données ... .  |
|   |  Chargement ...   |

<callout background="2">

Windows `alt + 0133`

macOS `option + ;`

</callout>

### Format des [nombres](https://prisme.lucca.io/94310e217/p/016f8d)

* Les milliers, millions, etc. sont séparés d’une espace insécable.
* Les nombres décimaux sont séparés d’une virgule.
* Les nombres cardinaux et ordinaux (quantité, distance, classement, hiérarchisation) s‘écrivent en chiffres.

| ✅ | ❌ |
| --- | --- |
| 12 345 678,90 | 12345678.90 |

<callout background="2">

En anglais, les milliers, millions, etc. sont séparés d’une virgule et les nombres décimaux sont séparés d’un point.

</callout>

#### Nombres ordinaux

Les nombres ordinaux doivent être utilisés grâce à la balise HTML `<sup>`.

#### Valeurs

* Un nombre en début de phrase s‘écrit toujours en toutes lettres.
* Les nombres inférieurs ou égaux à dix s‘écrivent en toutes lettres.
* Les valeurs (%, $…) sont toujours séparées de leurs unités par une espace insécable.
* Il ne faut pas utiliser le symbole des degrés pour faire un exposant o mais `<sup>o</sup>`

<callout background="3">

**Pour en savoir plus, lire "****[Nombres](https://prisme.lucca.io/94310e217/p/016f8d)****".**

</callout>

### Deux-points, listes

* Utilisons le signe deux-points pour introduire une liste à puces ou une série d'étapes. Ne l'utilisons pas à la fin des [titres](https://prisme.lucca.io/94310e217/p/63277d-ponctuation-typographie/t/751eb7).
* Les éléments de listes sont séparés par des points-virgules, des virgules ou des points. Ils commencent par une minuscule, sauf si ce sont des phrases.

| ✅ | ❌ |
| --- | --- |
| — J'ai besoin : – du nom, – du prénom, – du département, – et du site. | — J'ai besoin : – Du nom – Du prénom – Du département – Et du site |

### Ligatures

Les ligatures linguistiques, comme le "e dans l'o", sont obligatoires.

| ✅ | ❌ |
| --- | --- |
| Mon œuvre, æquo. Œuvre Æquo | Mon oeuvre, aequo. Oeuvre Aequo |

<callout background="2">

Raccourci pour le **œ** :

Windows `alt + 0156`

macOS `option + o`

Raccourci pour le **æ** :

Windows `alt + 145`

macOS `option + a`

Raccourci pour le **Œ** :

Windows `alt + 0140`

macOS `option + maj + o`

Raccourci pour le **Æ** :

Windows `alt + 146`

macOS `option + maj + a`

</callout>

La ligature du « & » est réservée aux noms commerciaux.

### Traits d'union, tirets et signes arithmétiques

* Le **trait d’union** (-) sert à unir (les mots, les lignes, les [dates](https://prisme.lucca.io/94310e217/p/301758) ou les plages horaires comme 9 h 30 - 18 h 30).

<callout background="2">

macOS : `Touche “Tiret”`

Windows : `Touche “6"`

</callout>

* Le **tiret moyen, ou demi-cadratin** (–) sert à signaler les changements d’éléments.

<callout background="2">

macOS : `option + shift + tiret`

Windows : `alt + 0150`

</callout>

* Le **tiret long, ou cadratin** (—) sert à signaler les changements de tons.

<callout background="2">

macOS : `option + tiret`

Windows : `alt + 0151`

</callout>

* Le **signe négatif** (−) sert à signaler une chiffre négatif et sera lu "moins" par le lecteur d'écran.

<callout background="2">

macOS : copier le signe `−`

Windows : `alt + 8722`

</callout>

* Les **signes plus (+) et moins (−) placés devant un nombre** suivent des règles d'espacement spécifiques.

| Signe | Avant le signe | Après le signe | Exemple |
| --- | --- | --- | --- |
| + | Une espace insécable | Pas d'espace | +359 collaborateurs |
| − | Une espace insécable | Pas d'espace | Il fait −3 °C. Un rendement de −17 %. |

**Attention :** pour les opérations arithmétiques (12 + 12 = 24 ou 87 − 66 = 21), on utilise une espace insécable avant **et** après les signes + et −.

### Lokalise

* Pas d’espace avant ou après les chaînes.
* Pas d’espace à l’intérieur des doubles accolades.
