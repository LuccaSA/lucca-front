# Content

## Principe général

**Liste à puces :** éléments de valeur égale (besoins, objectifs, ressources) 

**Liste numérotée :** étapes ou tâches dans un ordre spécifique

❌ Pas de sous-listes (trop complexes).

❌ Pas de liens hypertexte (lisibilité réduite).

---

## Règles de rédaction

| Aspect  | Règle |
| --- | --- |
| **Introduction** | Terminer par : deux-points, point d'interrogation, point, ou rien (si titre). |
| **Format** | Puce ou chiffre avant chaque élément |
| **Majuscule** | Minuscule si l'intro finit par : (complète la phrase) / Majuscule sinon - pour une phrase complète. |
| **Ponctuation** | Virgule si pas phrase complète. Point si phrase complète. Point final obligatoire en dernier élément. |
| **Structure** | 1 info = 1 élément. |
| **Mots-clés** | Placer au début de chaque élément. |
| **Longueur idéale** | 3 éléments (optimal). Au-delà de 6 : difficile à parcourir. |

**Bien noter :** chaque élément doit être cohérent avec les autres, avec la même formulation, le même niveau d'information.
