# Principes

## Nos polices de caractères

Nous utilisons trois polices de caractères complémentaires, chacune avec un rôle bien défini :

* **Lucca Sans :** c’est notre police de caractère maison. Nous l’utilisons pour les titres de niveau 1 et 2 mais aussi dès qu’il est intéressant d’apporter la Lucca’s touch dans nos interfaces.
* **Source Sans Pro :** la police par défaut pour le texte courant sur nos interfaces.
* **Caveat :** aussi utilisé par l’équipe Brand, cette police de caractère manuscrite apporte un ton plus humain lorsqu’on l’utilise dans des illustrations (Empty state par exemple).

### Configuration CSS

La police Caveat n'étant pas chargée par défaut, elle nécessite un import supplémentaire :

```css
@use '@lucca-front/scss/src/commons/config' with (
  $fontFamilyCursive: 'Caveat', /* v19.1 */
);
```

### Couleurs

Pour assurer cohérence et lisibilité, nous avons défini un ensemble de tokens spécifiques aux couleurs de typographie. Ils couvrent les usages principaux (titres, textes, éléments secondaires, etc.) et s’adaptent aux différents contextes d’interface.

L’utilisation de ces tokens permet une meilleure maintenabilité et une adaptation facilitée aux évolutions de notre Design System.

| Token | Valeur | Utilisation |
| --- | --- | --- |
| `--pr-t-color-text` | Neutral 800 | Couleur par défaut pour texte courant |
| `--pr-t-color-text-heading` | Neutral 900 | Titres de h1 à h6 |

## Règles d'utilisation

### Titres

Utilisons les styles de heading pour structurer les interfaces : le titre principal, les sous-titres pour introduire le contenu ou les sections. Ils aident les utilisateurs à parcourir rapidement l’information et à en comprendre la structure.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons la Lucca Sans pour les titres de niveau 1 et 2. |
| Don't |  |   | N’utilisons pas la Lucca Sans pour les titres de niveau 3 (et inférieur) ou les paragraphes. |

### Texte courant

Utilisons les styles body pour le contenu principal.

Ces styles s’utilisent après un titre ou un sous-titre, par exemple pour des descriptions ou des messages, mais peut aussi être utilisé seul dans les composants.

Dans Figma, l’espacement entre les paragraphes et les listes est défini dans les styles. Pour représenter des paragraphes dans le code, utilisez un composant texte distinct pour chaque paragraphe et gérez l’espacement.

### Lucca Sans

Lucca Sans est une police d’accent.

Elle doit être utilisée avec parcimonie pour mettre en valeur certains éléments de l’interface ou dans des composants précis du Design System. Son utilisation doit rester limitée pour conserver son rôle d’emphase et éviter toute surcharge visuelle.

| Rule | Image | Caption | Description |
| :--- | :--- | :--- | :--- |
| Do |  |   | Utilisons pas la Lucca Sans pour créer de l’emphase dans des composants spécifiques (Fancy box, Highlight data). |
| Don't |  |   | L’usage répété de Lucca Sans dans une même interface ou dans un même composant réduit son impact et brouille la hiérarchie visuelle. |
| Do |  |   | Utilisons la Lucca Sans lorsqu’elle est intégrée nativement dans un composant du Design System. Son usage y est cadré pour apporter de l’emphase tout en garantissant la cohérence visuelle globale. |
| Don't |  |   | Ne surchargeons pas les composants du Design System avec des styles typographiques personnalisés. Cela nuit à la cohérence et complique la maintenance. |

### Longueur des paragraphes

Un bloc de texte ne doit être ni trop court ni trop long pour garantir une bonne lisibilité. Pour cela, nous recommandons de ne pas dépasser 80 caractères par ligne.

### Alignements

Le texte doit être ferré à gauche pour des questions de lisibilité. De la même manière, il n’est jamais justifié.

Dans les tableaux, les nombres doivent être alignés à droite pour faciliter la lecture et la comparaison des données numériques. Il est nécessaire d’afficher le même nombre de décimales pour ne pas perturber la lecture, quand bien même celles-ci sont “,00”.

Il existe une propriété CSS pour garantir un alignement des nombres :`font-variant-numeric: tabular-nums;`

### Utilisation des capitales

L’utilisation des capitales (petites capitales inclues) est proscrite, mis à part pour des acronymes.
